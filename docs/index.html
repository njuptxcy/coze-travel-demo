你现在看到的重点

右侧聊天面板保留 Coze 官方 SDK 的原生交互能力，外层容器则由这个网页控制，所以现在可以拖动、复位，也能继续改页面视觉。

核心能力

这不是大而全的介绍页，而是把这个项目最适合展示给老师、面试官和自己复盘的内容挑出来放在一页里。

适合 Demo 展示

出差申请

收集出发地、目的地、时间、事由和交通方式，再由工作流完成申请表单落库。

费用标准匹配

结合城市分类、职级标准和出差时长，估算住宿、市内交通和补贴金额。

知识库检索

先召回城市、机场、费用项目等候选数据，再由大模型做二次提取和结构化输出。

卡片交互

把申请信息整理成更清晰的确认视图，减少用户看漏字段、填错字段的概率。

现在这套架构怎么理解

用最直白的话说：网页只是壳，真正的智能体能力主要还在 Coze 云端。

适合讲项目

1. 页面层

当前网页负责视觉展示、说明文案、布局结构，以及把聊天窗口放进一个更像产品的界面里。

2. 交互接入层

Chat SDK 相当于桥，把网页和你发布到 Coze 的 Agent 接起来，同时保留卡片、按钮和多轮对话能力。

3. 智能体能力层

意图识别、工作流路由、知识库检索、数据库写入和卡片动作，核心都发生在 Coze 侧。

4. 后续扩展层

如果以后要做更像企业系统的版本，可以把现在这个前端保留，再补一层你自己的后端去做鉴权、日志和业务系统对接。

推荐展示流程

你后面给老师或面试官演示时，可以按这个顺序讲，逻辑会更顺。

适合答辩
1

先讲目标

这是一个企业商旅费控智能体 Demo，核心目标是把出差申请、标准匹配、差旅信息查询和数据落库串起来。

2

再讲 Coze 复刻过程

说明你是从已有成品出发，自己重新搭建知识库、数据库、工作流、卡片和提示词，而不是只会点几下现成模板。

3

最后讲产品化包装

这个网页就是你把 Agent 从平台内部使用，包装成可以公开演示的外部产品页面的第一步。
